# AMNLT models

A collection of models that can be used for [AMNLT dataset](https://huggingface.co/datasets/PRAIG/AMNLT).

## Generating datasets

Part of this work is focused on unifying encoding of all of the datasets presented in AMNLT paper. It is not clear if the datasets can be redistributed so this repository provides a way to generate datasets with unified encoding given that you have access to the huggingface datasets.

The original AMNLT datasets contain duplicates (even cross split contamination!) and samples that are rendered incorrectly (both of these problems are almost exclusive to `GregoSynth_staffLevel` dataset). To mitigate this deduplication pipeline was constructed (inside `deduplication/` directory) and the incorrectly rendered samples are mostly filtered by strict grammar checking of the transcriptions. 

To generate all datasets, run the `generate_datasets.sh` script with installed python dependencies. But keep in mind that it is **resource intensive**. Minimum requirements are 16+GB of free RAM, 64GB of disk space (preferably NVMe SSD) + space for the original datasets (16GB). It is also compute intensive (mainly the deduplication), it takes ~45 minutes on Intel Core Ultra 7 265k using all 20 threads.

Output structure:

- `conversion_to_common.log` - outputs of GabcParser/gabcparser/utils/common_encoding.py (mainly list of samples that cannot be parsed - many of them are rendered incorrectly)
- `conversion_validation.log` - outputs of GabcParser/gabcparser/utils/grammar_validation.py which validates the conversion
- `datasets/` - generated datasets
  - `common_encoding/` - datasets converted to common encoding - without any deduplication
  - `common_encoding_deduplicated/` - deduplicated datasets from `common_encoding/`
  - `deduplicated_original/` - original datasets filtered so that only contain same samples as the `common_encoding_deduplicated/` datasets
- `deduplication_files/` - temporary files generated by deduplication pipeline - they can be useful for further analysis of the datasets
  - `duplicates_*.json` - indices of all duplicate samples
  - `filtered_indices_*.json` - indices that were filtered out due to being duplicates
  - `sim_*_common.bin` - upper triangular part of the similarity matrix for given dataset (only for `common_encoding/` datasets) - similarity metric is Levenshtein distance (computed using [StringZilla](https://github.com/ashvardanian/stringzilla))
  - `sim_*_common.bin.json` - accompanying data to the similarity matrix - needed for proper usage of the similarity matrix
